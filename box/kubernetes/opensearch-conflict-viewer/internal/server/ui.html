OpenSearch Mapping Conflicts

Common root cause: when a new daily index is created without an explicit mapping (index template), dynamic mapping picks the field type from the first document ingested that day. The type gets re-rolled every day, and querying through an index pattern surfaces the mismatch as a conflict.

Fix: pin the field mapping in an index template, or normalize the value type before ingestion (e.g. in Logstash). Existing indices keep the conflict until reindexed; daily indices age out with retention.
